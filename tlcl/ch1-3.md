# TLCL Notes — Chapters 1–3

## Chapter 1 — What Is the Shell?

### Overview
The shell is a program that accepts commands from the keyboard and passes them to the operating system.  
In most Linux systems, the default shell is `bash` (*Bourne Again SHell*).  
A terminal emulator provides a graphical window for interacting with the shell.

### Key Points
- The prompt indicates that the shell is ready to accept input.
- A typical prompt usually includes:
  - username
  - hostname
  - current directory
  - `$` for a regular user
- A prompt ending with `#` usually indicates root privileges.
- `command not found` means the shell could not locate the requested command.

### Basics
- The shell keeps a command history.
- Previous commands can be recalled with the Up Arrow key.
- Commands can be edited directly on the command line before execution.

### Summary
This chapter introduces the shell as the main command-line interface in Linux, the role of the terminal emulator, and the basics of entering and recalling commands.

## Chapter 2 — Navigation

### Overview
Linux organizes files and directories in a single hierarchical tree structure that begins at `/`, the root directory.

### Core Commands
- `pwd` — print the current working directory
- `ls` — list directory contents
- `cd` — change the current directory

### Key Concepts
- **Current working directory** — the directory currently in use
- **Home directory** — the default directory assigned to a user

### Paths
- **Absolute path** — starts with `/` and defines the full path from the root  
  Example: `/usr/bin`
- **Relative path** — defines a path from the current directory

### Special Symbols
- `.` — current directory
- `..` — parent directory

### Common `cd` Usage
- `cd` — move to the home directory
- `cd -` — move to the previous directory
- `cd ~user` — move to another user's home directory

### File Naming Notes
- Names beginning with `.` are hidden files.
- Linux is case-sensitive.
- File extensions are less important than in Windows.
- Spaces in file names are allowed, but `_` is usually preferred.

### Summary
This chapter introduces the basic navigation workflow in Linux:
- identify the current location with `pwd`
- inspect contents with `ls`
- move through the filesystem with `cd`

## Chapter 3 — Exploring the System

### Overview
This chapter introduces three commands used to inspect and explore the system:
- `ls`
- `file`
- `less`

### `ls` — List Files and Directories
The `ls` command displays directory contents and can also show detailed file information.

#### Common Options
- `-a`, `--all` — show all files, including hidden files
- `-A` — show almost all files except `.` and `..`
- `-d` — show the directory itself, not its contents
- `-F` — append file type indicators
- `-h` — show human-readable file sizes
- `-l` — use long listing format
- `-r` — reverse sort order
- `-S` — sort by file size
- `-t` — sort by modification time

#### `ls -l` Output Includes
- file type and permissions
- number of hard links
- owner
- group
- file size
- modification time
- file name

### `file` — Determine File Type
The `file` command identifies the actual type of a file based on its contents.

#### Example
```bash
file somefile

### `less` — Read Text Files

The `less` command displays text files one page at a time.

#### Example

```bash
less /etc/passwd

#### Basic Keys

- `Space` / `Page Down` — move forward
- `b` / `Page Up` — move backward
- `g` — go to the beginning
- `G` — go to the end
- `/text` — search forward
- `n` — go to the next match
- `h` — open help
- `q` — quit

### Common Directories

- `/` — root directory
- `/bin` — essential user commands
- `/boot` — boot files
- `/dev` — device files
- `/etc` — system configuration files
- `/home` — user home directories
- `/lib` — shared libraries
- `/media` — removable media mount points
- `/mnt` — temporary mount points
- `/opt` — optional software
- `/proc` — process and kernel information
- `/root` — root user home directory
- `/sbin` — system binaries
- `/tmp` — temporary files
- `/usr` — user programs and data

### Summary

This chapter introduces the basic tools for inspecting the Linux system:

- `ls` for listing files and attributes
- `file` for identifying file types
- `less` for reading text files and configuration files
