# Linux Upskill

Учебный репозиторий по Linux с упором на практику, системное мышление и аккуратную фиксацию прогресса.

Здесь я собираю:
- конспекты по дням **Linux Upskill Challenge**;
- краткие **cheatsheets** по основным темам;
- небольшие практические **labs**;
- заметки по книге **The Linux Command Line**;
- общий **logbook** с ходом обучения.

## Цели проекта

- закрепить базу по Linux через ежедневную практику;
- научиться увереннее работать с сервером по SSH;
- систематизировать команды, которые реально нужны в работе;
- развить привычку документировать эксперименты и наблюдения;
- собрать аккуратное портфолио учебного прогресса по Linux.

## Что уже есть в репозитории

Сейчас в репозитории есть основные учебные разделы:
- `days/` — заметки по дням Linux Upskill Challenge;
- `cheatsheets/` — короткие шпаргалки по темам;
- `labs/` — маленькие практические эксперименты;
- `tlcl/` — заметки по книге *The Linux Command Line*;
- `logbook.md` — общий журнал прогресса.

На данный момент в `days/` уже есть `day0.md` и `day1.md`, в `tlcl/` — конспект `ch1-3.md`, а в остальных папках подготовлена структура и README под дальнейшее наполнение.

## О чем этот проект по содержанию

### 1. Linux Upskill Challenge
В разделе `days/` я веду заметки по прохождению Linux Upskill Challenge.

Уже зафиксированы:
- настройка SSH-доступа к облачной Ubuntu VM;
- генерация отдельного SSH-ключа;
- базовые меры по безопасным правам доступа на ключи;
- первичная инвентаризация сервера;
- команды для проверки ОС, железа, нагрузки, дисков и сети.

### 2. Cheatsheets
Папка `cheatsheets/` предназначена для коротких справочных материалов по темам, к которым удобно возвращаться перед практикой.

Планируемые темы:
- paths и navigation;
- file permissions;
- SSH;
- processes;
- systemd;
- networking;
- grep/find;
- pipes и redirection.

Формат шпаргалок простой: что это такое, какие команды нужны чаще всего и какие ошибки встречаются чаще всего.

### 3. Labs
В `labs/` лежат небольшие практические упражнения.

План репозитория уже предполагает лабораторные заметки по таким темам:
- права доступа к файлам;
- пользователи и группы;
- grep/find;
- pipes и redirection;
- systemctl и логи;
- базовая проверка сети.

Для каждого lab-файла я хочу сохранять:
- цель;
- использованные команды;
- результат;
- краткий вывод — что именно удалось понять или закрепить.

### 4. TLCL Notes
Раздел `tlcl/` содержит мои заметки по книге *The Linux Command Line*.

Сейчас уже есть конспект по главам 1–3:
- что такое shell;
- навигация по файловой системе;
- базовое исследование системы через `ls`, `file`, `less`;
- ключевые понятия вроде prompt, absolute/relative path и скрытых файлов.
